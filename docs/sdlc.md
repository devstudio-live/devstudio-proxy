# devstudio-proxy SDLC & Workflow

## Branching & Commit Conventions
- Follows the same conventions as devstudio (see devstudio/CLAUDE.md)
- Feature branches: `feature/<name>`
- Release branches: `release/<version>`
- Commit pairing: feature + docs update

## Release Process
- Releases are tagged and documented in devstudio/ and devstudio-proxy/
- Major plans and trackers live in devstudio/plans/ and are referenced here

## Contribution Workflow
- All changes require code review
- Use PR templates and reference relevant plans
- Cross-repo changes coordinated via devstudio as master repo

## Documentation
- Architectural and SDLC docs live in devstudio-proxy/docs/
- Reference devstudio/CLAUDE.md for shared rules
